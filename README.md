Electronics Shopping App – Manual Testing
1. Project Title
Electronics Shopping Web Application – Manual Testing
2. Project Overview
This project involves manual testing of an Electronics Shopping Web App, where users can browse, search, and buy electronic items such as mobiles, laptops, and accessories. The goal of this testing is to ensure all features work correctly and users have a smooth shopping experience.
3. Features Tested
•	User Registration
•	Login Module
•	Categories Feature
•	Products & Shopping cart 
•	Checkout & Payment
•	Contact us
4. Testing Type
•	Functional Testing
•	Validation Testing
•	System Testing
5. Tools Used
•	Microsoft Excel – For RTM, writing test cases,test data and tracking execution status
•	Microsoft Word – For Test Plan and Summary Report documentation
•	Google Chrome & Microsoft Edge – For testing the web app
6. Test Artifacts
•	Test Plan
•	RTM Document
•	Test Cases
•	Test Data
•	Defect Report
•	Test Summary Report
7. Test Execution Summary
•	Total Test Cases: 65
•	Executed: 65
•	Passed: 57
•	Failed: 8
•	Defects Raised: 8
8. Defects Found
•	Registration: Validation issues in Name and Password fields
•	Login: Incorrect/missing validation for email and password
•	Checkout: Multiple shipping and payment options not displayed.
9. Learning Outcomes
•	Practical understanding of Manual Testing
•	Writing and executing manual test cases
•	Logging defects accurately
